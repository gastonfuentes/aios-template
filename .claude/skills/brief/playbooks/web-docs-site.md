# Playbook: web-docs-site

## Targets obligatorios
- **Comparar frameworks**: Fumadocs vs Nextra vs Starlight vs Docusaurus (features, perf, mantenimiento).
- **Busqueda**: Algolia DocSearch (gratis para OSS), flexsearch (client-side), typesense self-hosted.
- **Versionado**: como se gestionan docs de multiples versiones de producto.
- **Estructura de docs**: Diataxis (tutorials / how-to / reference / explanation).

## Targets opcionales
- **i18n** si la doc es multi-idioma.
- **Edit on GitHub** boton por pagina para contribuciones.
- **Feedback widget** por articulo ("¿Fue util?").

## Busquedas sugeridas
- "Fumadocs vs Nextra 2026"
- "Diataxis documentation framework"
- "Algolia DocSearch OSS application"

## Fuentes primarias
- https://diataxis.fr
- https://fumadocs.vercel.app
- https://docsearch.algolia.com

## Riesgos a investigar activamente
- Documentacion vs blog: evitar mezclar conceptos, pueden confundir SEO.
- Mantenimiento long-term: ¿quien actualiza las docs cuando cambia el producto?
